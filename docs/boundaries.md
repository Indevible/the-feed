# Boundaries

The Feed is a public interpretation layer for public XDEL activity.

## The Feed does

- observe public blockchain activity;
- apply a published numeric grammar;
- produce normalized Feed events;
- resolve readable names where appropriate;
- link reciprocal interactions;
- present recent activity on community surfaces;
- provide links back to source transactions.

## The Feed does not

- hold keys;
- request signatures;
- move XDEL;
- control wallets;
- automate treasury activity;
- promise market outcomes;
- score personal worth;
- create negative reputation records;
- prove private intent;
- define durable Vouch mechanics.

## Interpretation is not proof of intent

A numeric pattern may match a published code even when the sender did not intend to use the language. Feed output should be framed as an interpretation of public activity, not a factual claim about private motivation.

## No social obligation

An open gesture does not create a duty to respond.

Unanswered activity may scroll away naturally. It should not be labeled rejected, failed, ignored, or incomplete in a way that pressures the recipient.

If a valid reciprocal response occurs later, the completed interaction may reappear as new Feed activity.

## No negative ledger

The Feed is not a blacklist, accusation system, cancellation archive, or permanent negative score.
