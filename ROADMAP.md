# Roadmap

## M0 — Public design

- Define the boundary of The Feed.
- Define provisional terminology.
- Define the numeric transaction grammar.
- Define the normalized Feed event model.
- Document community integration requirements.
- Draft the first small code lexicon.

## M1 — Offline interpreter

- Parse sample XDEL amounts.
- Separate economic value from appended expression.
- Classify wallet, swap, and trade contexts.
- Produce normalized Feed events.
- Test collision and ambiguity rules.

## M2 — Read-only chain activity

- Observe canonical XDEL activity.
- Resolve readable wallet names where available.
- Detect provisional codes.
- Link reciprocal activity.

## M3 — Public web Feed

- Display recent interpreted activity.
- Resurface completed reciprocal interactions.
- Link each event to its source transaction.
- Create stable, shareable event URLs.

## M4 — Community outputs

- Discord webhook or bot output.
- Telegram bot or channel output.
- Configurable presentation templates.
- Community-specific filters without changing the canonical event.

## M5 — Meme renderer

- Deterministic visual templates.
- Code-specific captions and iconography.
- Shareable images and short animations.
- Controlled output suitable for public channels.

## M6 — Public integration layer

- Read-only API.
- Event stream or webhook delivery.
- Embeddable Feed component.
- Documentation for outside communities.

## Outside this repository

Durable Vouch mechanics, standing, identity permanence, custody, treasury operations, and governance over personal recognition are separate Indelible systems.
